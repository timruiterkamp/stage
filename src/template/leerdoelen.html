Leerdoel 1

Als eerste leerdoel heb ik gesteld dat ik binnen een periode van 2 maand m n code kwaliteit wil verbeteren. Deze code moet goed z n opgebouwd, herbruikbaar z n en worden opgeslagen doormiddel van versie controle. Ik vind het belangr k dat m n code goed wordt opgebouwd en herbruikbaar is om zo b overdrachten niet een andere developer hoef uit te leggen waar alles staat., ook voorkom je hiermee dat code snel kapot zal gaan. Om dit leerdoel te halen heb ik m verdiept in BEM, SMACSS, Linters, Git en de front-end repository van het bedr f zelf. Doormiddel van de front-end repository te gebruiken leerde ik snel hoe ik m n code kon opbouwen zodat het voor iedereen te benaderen en te begr pen is. Hiernaast heb ik veel onderzoek gedaan in de eerste maand naar BEM, SMACSS en de front- end repository. Door dit toe te passen op test projecten kwam ik er snel achter dat het erg handig is en je op een snellere en betere manier je code kunt schr ven en opbouwen. Nadat ik m verdiept had in deze manier van werken ging ik aan de slag met linters en versiebeheer. Hierdoor had ik een betere grip op m n werk en kwam ik er al snel achter dat ik fouten maakte b het schr ven van m n code. Doormiddel van de linters verbeterde ik al snel fouten die ik vaak maakte omdat ik telkens een error kreeg en deze moest oplossen/verbeteren. Met het gebruik maken van versiebeheer ging er voor m toch weer een wereld open. Ik kende Git al voordat ik stage ging lopen en heb er een aantal keer naar gekeken, maar nu leerde ik er echt mee werken en te begr pen. In deze maand en zeker ook de maanden die hierop volgde heb ik met hulp van collega’s m n weg gevonden naar het goed omgaan met versiebeheer.
