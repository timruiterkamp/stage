Leerdoelen

Vanuit school is de opdracht meegegeven om doelen voor jezelf te stellen. Dit is omdat je zo ergens naar toe kunt werken en ook kunt controleren of je het hebt gehaald of niet. Voor de stage had ik zoals eerder aangegeven een doel om een betere front-end developer te worden, dit is ook de richting waar ik in wil gaan en later mijn beroep van wil maken. Een betere front-end developer worden is alleen een erg breed begrip dus mijn doelen heb ik kleiner gemaakt en verdeeld in twee leerdoelen. Uiteindelijk ben ik tot de conclusie gekomen dat ik mij vooral verder wil ontwikkelen op het gebied van Javascript, Javascript frameworks als AngularJS en code kwaliteit. Voor mijn gevoel is hier nog een hele stap te zetten voor mij.

Leerdoel 1

Als eerste leerdoel heb ik gesteld dat ik binnen een periode van 2 maand mijn code kwaliteit wil verbeteren. Deze code moet goed zijn opgebouwd, herbruikbaar zijn en worden opgeslagen doormiddel van versie controle. Ik vind het belangrijk dat mijn code goed wordt opgebouwd en herbruikbaar is om zo bij overdrachten niet een andere developer hoef uit te leggen waar alles staat, ook voorkom je hiermee dat code snel kapot zal gaan. Om dit leerdoel te halen heb ik mij verdiept in BEM, SMACSS, Linters, Git en de front-end repository van het bedrijf zelf. Doormiddel van de front-end repository te gebruiken leerde ik snel hoe ik mijn code kon opbouwen zodat het voor iedereen te benaderen en te begrijpen is. Hiernaast heb ik veel onderzoek gedaan in de eerste maand naar BEM, SMACSS en de front-end repository. Door dit toe te passen op test projecten kwam ik er snel achter dat het erg handig is en je op een snellere en betere manier je code kunt schrijven en opbouwen. Nadat ik mij verdiept had in deze manier van werken ging ik aan de slag met linters en versiebeheer. Hierdoor had ik een betere grip op mijn werk en kwam ik er al snel achter dat ik fouten maakte bij het schrijven van mijn code. Doormiddel van de linters verbeterde ik al snel fouten die ik vaak maakte omdat ik telkens een error kreeg en deze moest oplossen/verbeteren. Met het gebruik maken van versiebeheer ging er voor mij toch weer een wereld open. Ik kende Git al voordat ik stage ging lopen en heb er een aantal keer naar gekeken, maar nu leerde ik er echt mee werken en te begrijpen. In deze maand en zeker ook de maanden die hierop volgde heb ik met hulp van collega’s mijn weg gevonden naar het goed omgaan met versiebeheer.

Leerdoel 1 reflectie

In het begin toen ik onderzoek ging doen naar BEM, SMACSS en de front-end repository had ik het even moeilijk met opgang komen en te weten waar ik precies moest zoeken. Nadat ik wat meer uitleg vroeg bij collega’s kreeg ik het snel door en leerde ik de gemakken kennen van het werken met deze methodes. Ik ben blij dat ik onderzoek heb gedaan en heb leren om te gaan met zowel de manieren van het opbouwen en schrijven, als met het leren gebruik te maken van versiebeheer. Al deze nieuwe methodes hebben ervoor gezorgd dat ik zelf merk dat mijn code makkelijker te lezen is en ook gemakellijker in te zetten is bij andere projecten. Gedurende mijn stageperiode heb ik zelf ook gemerkt dat wanneer ik bijvoorbeeld mijn code liet controleren bij een begeleider het er vaak wel goed uitzag. Hooguit een paar puntjes die op een andere manier netter zou kunnen worden opgelost. Met de kennis die ik nu heb over de verschillende manieren van code schrijven, opbouwen en beheren kan ik stellen dat ik mijn leerdoel heb gehaald.
